Internal Global Support Services and IVR Usability Recommendations

Problems

The internal support chatbot was outdated and had redundant use cases. It needed to be redesigned to provide better support to a new audience.

There are legacy voice bots that needed to be analyzed for usability improvement

Results

Replicated, edited and created new use cases for internal chatbot for increased usability for users

Used data analysis on legacy voice bots to identify areas of improvement for usability


Internal Chatbot- Project Scope

The company decided to redesign the current internal support chatbot for customers. The current support chatbot contained use cases that were outdated and redundant. Stakeholders believed the chatbot wasn’t supporting customers’ needs. The chatbot also had new use cases that needed to be included within the bot.

User could use the old system within the internal help center, but there wasn’t enough traction from them. Stakeholders decided the bot was needed to be redesigned.

UX Design Process

In this case, I had no direct access to customers. I worked with my supervisor. Together we did the following:

Communicated with stakeholders on each use case. Figured out which use cases to keep. Some use cases were meant for older products

Replicated, edited and created new use cases for the both. Some use cases were replicated from a similar bot with another customer group. However, they were a different set of products with different use cases.

Reviewed use case design with supervisor. The idea is to make sure that each use case was designed properly.

It should be noted that my designs were used in the second stage of the project. They were still in the rough draft stage. While I can’t show the exact design, due to an NDA, I can show parts of it.

Diagram of Chatbot

Our team also attempted to integrate the company’s generative AI through their APIs. The idea was to introduce customers to it for any use cases that didn’t exist. The API provided high level answers regarding products by absorbing data from existing knowledge bases, product articles, etc. I helped my team do the following:

Exploring the existing APIs capabilities to understand which parts were important.

Defining the most important parts of the API. We ended up exploring the ticket number, the token count returned to the customer, the initial personality prompt provided within the API.We also integrated a few parts of the generative AI with the older chatbot as well.

Voicebot Usability Recommendations

As a consulting company, Avaya worked with different companies for different use cases. I got to work on two different voice bots that were considered legacy systems. Their recognition systems were very old.

The first project was for an airlines. I reviewed their transcripts to understand which utterances/words were recognized by the system and which ones weren’t. I did this for the crew voicebot and their passenger voicebot. I ended up providing the following recommendations:

Adding possible synonyms to existing use cases to help the bot recognize users’ utterances more

Adding more time for accepting possible utterance responses

Eliminating utterances that were similar with different use cases.

Another project I worked was a voicebot for a utilities company. I helped test out street names in different languages. I was able to identify which names weren’t recognized by the system.

Limitations

For the UX designs, there were situations where the transfer protocols for certain languages. Additionally, we had to make sure the generative AI and the current chatbot were working together. The voicebots I worked on were legacy systems and didn’t have up to date software.

Conclusion

Overall, I learned a lot from this experience. I worked with tools like Nuance Transcription and parseTool for the voicebots. I also got to explore different AI tools like Cognigy and Google Dialogflow more in depth for the new bot. Learning how to design was an amazing experience as I got to work with old style chatbots with generative AI. Overall, there’s still a lot more I wish I did.
