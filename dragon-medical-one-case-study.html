Dragon Medical One-Nuance Communications

Project Context

Dragon Medical One is a voice transcribing tool doctors can use to record patient notes. It’s also a leading voice dictation tool for users who don’t have the ability to write, type, etc. For Dragon Medical One, the user interfaces needed to be catalogued for future designers and engineers in the company. The product also needed a user analysis/heuristic evaluation to identify any usability painpoints.

Problem

Dragon Medical One needed its UI catalogued for future designers and engineers. The tool also needed a user analysis to find usability painpoints in the system.

Solution

Catalogue Dragon Medical One UI features in some organized manner and conduct a user analysis/heuristic analysis on Dragon Medical One

User Analysis

While documenting Dragon Medical One's UIs, I initiated a user analysis to identify potential areas for system enhancement. Adopting the perspective of a new user unfamiliar with medical transcription tools, I meticulously traversed through the entire system. I maintained a personal log to record every identified pain point, mirroring the breakdown utilized for the UI catalog. By using Norman Nielsen's heuristics, I categorized each problem area and then applied a rating system to assess the criticality of each pain point.

  • Cosmetic Issue
  • Minor Issue
  • Major Issue
  • Critical Issue

For the final deliverable, I made a web page using a tool called Microsoft Sway. The webpage was broken into every section as listed above. Each section included the following:

  • Screenshot of painpoint
  • Description of painpoint
  • Heuristic for the related painpoint
  • Level of severity for painpoint
  • Explanation about painpoint

Screenshot of User Analysis

Some of the results were the following:

  • Icon recognition was easy to follow (Lightbulb icon)
  • There were unnecessary pages/tools users had to navigtate and their branding/style was inconsistent across every part
  • More intutitive terminology could be used to provide more feedback
  • Certain pages needed to be prioritized more.
  • Pages/features needed to be rearranged to be combined with each other

Limitations

During the Dragon Medical One user analysis, my focus shifted from highlighting pain points solely from the user's perspective to identifying them from the viewpoint of a new user encountering the device for the first time.

Conclusion

Before this internship, I had little knowledge about medical dictation software. However, through my engagement with the product, I discovered its versatility beyond the medical realm. This experience illuminated my interest in further exploring and working with such technology in the future. Additionally, this internship reaffirmed my aspiration to pursue a career in the medical field moving forward.
